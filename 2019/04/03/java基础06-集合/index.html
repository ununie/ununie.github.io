发表于 2019-04-03

1、集合

数组长度固定，不能满足长度变换的要求，所以出现了集合。

集合和数组的区别

  • 长度区别：数组长度固定，集合长度可变
  • 存储内容不同：数组只能存储单一数据类型，集合可以存储不同数据类型
  • 数组可以存储引用类型和基本类型，集合只能存储引用类型。

2、List

List是Collection的子接口

并发修改异常

迭代器迭代集合，集合修改元素会出现并发修改异常

解决方案：

迭代器遍历，迭代器修改。使用ListIterator迭代器，元素添加在迭代器修改的位置。

集合遍历，集合修改size()和get()，元素添加在末尾。

常见的数据结构

栈：先进后出

队列：先进先出

数组：查询快，增删慢

链表：查询慢增删快

List子类特点

ArrayList：底层是数组，查询快，增删慢。线程不安全，效率高

Vector：底层是数组，查询快，增删慢。线程安全，效率低

​ 即使需要安全考虑也不使用Vector

LinkedList：底层是链表，查询慢，增删快。线程不安全，效率高

5、Set

不包含重复元素，无序集合。它有自己的存储顺序

1、HashSet

底层是HashMap

2、LinkedHashSet

具有可预知迭代顺序的Set接口实现。

底层数据结构由哈希表和链表组成。哈希表保证唯一，链表保证有序。

3、TreeSet

底层是TreeMap。使用元素自然顺序对元素进行排序，或根据创建set时提供的Comparator进行排序，具体取决于构造方法。

TreeSet底层是二叉树结构（红黑树，是一种自平衡二叉树）

特点：排序、唯一

自然排序：进行自然排序的类实现了Comparable接口。调用Comparable接口的compareTo()方法进行比较后排序。

比较器排序：让集合的构造方法接受一个比较器Comparator接口的实现类对象，一般是使用匿名内部类的方式重写compareTo()方法

根据比较的返回值是否为0来决定元素的唯一性。

6、Map

1、HashMap跟Hashtable的不同

Hashtable不允许使用null键null值，是同步的，效率低

HashMap允许使用null键null值，不同步的，效率高
